# Acceptance

- [ ] Codex exec uses `-a` and `-s` instead of `--full-auto`.
- [ ] Empty system config values default to `on-request` and `workspace-write`.
- [ ] Custom system config values are forwarded to the Codex command line.
- [ ] Invalid approval or sandbox values fail before Codex is launched.
- [ ] Execution logs record the effective approval and sandbox values.
- [ ] `go test ./...` passes.
